## Требования

Процесс регистрации на сайте должен включать следующие шаги:

1. Пользователь открывает страницу регистрации.
2. Пользователь заполняет обязательные поля: имя, фамилия, адрес электронной почты, пароль и подтверждение пароля.
3. Система проверяет корректность введённых данных (формат email, совпадение паролей, минимальная длина пароля).
4. Пользователь соглашается с условиями использования (чекбокс).
5. Пользователь нажимает кнопку «Зарегистрироваться».
6. Система отправляет письмо с подтверждением на указанный адрес электронной почты.
7. Пользователь переходит по ссылке из письма для подтверждения аккаунта.
8. После подтверждения аккаунт считается активированным, и пользователь получает доступ к системе.

## Нефункциональные требования

- **Производительность:** страница регистрации должна загружаться не более чем за 2 секунды.
- **Безопасность:** пароль пользователя должен храниться в зашифрованном виде (хэш + соль). Передача данных осуществляется только по протоколу HTTPS.
- **Надёжность:** письмо с подтверждением должно доставляться в течение 1 минуты в 99% случаев.
- **Удобство использования:** форма регистрации должна корректно отображаться на мобильных устройствах и в актуальных браузерах.
- **Доступность:** интерфейс регистрации должен соответствовать стандарту WCAG 2.1 уровня AA.
- **Масштабируемость:** система должна выдерживать одновременную регистрацию не менее 1000 пользователей без деградации производительности.
